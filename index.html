Persistent memory for AI coding agents

Gives Claude Desktop, Cursor, Hermes, and Copilot a tiered memory store that persists across sessions. Agents load context automatically on startup and write corrections as they learn.

bash
curl -fsSL https://ai-memory-booster.com/install.sh | bash

macOS · Linux · Windows  ·  source  ·  MIT

What it does

AI agents lose all context when a session ends. Memory Enhancer Pro creates a local directory at ~/self-improving/ with three memory tiers — hot, warm, and cold — and installs agent configuration files that tell the agent to read from and write to those tiers on every session.

Installation takes under 30 seconds. No server, no account, no telemetry. Memory files are plain markdown you can read and edit directly.

How it works

  1. Memory tiers created

    Installer creates ~/self-improving/{hot,warm,cold,learnings}/. Hot holds the last 7 days, warm the last 30, cold is permanent. An hourly heartbeat promotes files between tiers automatically.

  2. Agent configuration installed

    SOUL.md and AGENTS.md tell the agent to load hot context at session start, cross-reference warm for recent decisions, and append corrections to learnings/corrections.md when you correct it.

  3. Platform integration

    On macOS/Linux the installer detects Claude Desktop, Cursor (.cursor/rules/), Hermes, OpenClaw, and Cline and wires each one in. Cloud backup runs optionally via the sync service in the background.

  4. Restart your agent

    That's it. The agent will load ~/self-improving/SOUL.md on the next session start and begin reading and writing memory automatically.
